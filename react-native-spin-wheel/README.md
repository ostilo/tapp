# react-native-spin-wheel

React Native wrapper around the Kotlin `android-widget` spin wheel library.

This package exposes a single Fabric‑ready component:

```tsx
import {SpinWheelView} from 'react-native-spin-wheel';

<SpinWheelView configUrl="https://example.com/tapp_widget_config.json" />;
```

Under the hood it bridges to `com.tapp.spinwheel.SpinWheelView` and uses
React Native's codegen (`codegenNativeComponent`) to generate the view config.

> **Note**  
> In this interview repo the Android implementation depends on a sibling Gradle
> module `:android-widget`. When using this package outside the monorepo, you
> either need to mirror that setup or inline/publish the widget module as a
> standalone AAR.

## Project layout

- JS/TS:
  - `src/specs/SpinWheelNativeComponent.ts` – Fabric component spec used by codegen.
  - `src/index.ts` – public JS entry point exporting `SpinWheelView`.
  - `lib/**` – compiled JS + declaration files (generated by `npm run build`).

- Android:
  - `android/src/main/java/com/tapp/spinwheel/rn/SpinWheelViewManager.kt`
    - Fabric view manager that bridges the `configUrl` prop to the Kotlin view.
  - `android/build.gradle`
    - Applies `com.facebook.react` plugin and depends on the shared `android-widget`
      module in this monorepo.

## Installation (inside this monorepo)

The `demo-app` is already wired to use this module. The relevant lines in
`demo-app/android/settings.gradle`:

```gradle
include ':react-native-spin-wheel'
project(':react-native-spin-wheel').projectDir = new File(rootProject.projectDir, '../react-native-spin-wheel/android')

include ':android-widget'
project(':android-widget').projectDir = new File(rootProject.projectDir, '../android-widget')
```

Then from `demo-app`:

```bash
npm install
npm run android
```

## Using the .tgz (JS wrapper) in another app

You can generate a tarball for the JS wrapper:

```bash
cd react-native-spin-wheel
npm run build
npm pack
```

This will create `react-native-spin-wheel-1.0.0.tgz`. In another React Native app:

```bash
npm install /absolute/path/to/react-native-spin-wheel-1.0.0.tgz
```

However, the Android native side still expects the `:android-widget` Gradle project. To
make this completely standalone for production you would:

1. Publish `android-widget` as a Maven/AAR dependency and depend on it by coordinate.

For the purposes of this assignment, the supported integration is via the monorepo
setup described above.
